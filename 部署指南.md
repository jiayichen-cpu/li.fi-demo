# 部署到生产环境指南

## 方法一：通过 Vercel（推荐，最简单）

### 步骤 1：初始化 Git 仓库

```bash
cd /Users/yael/Downloads/lifi-carnival-landing

# 初始化 Git
git init

# 添加所有文件
git add .

# 提交代码
git commit -m "Initial commit: LI.FI Trading Carnival landing page"
```

### 步骤 2：推送到 GitHub

1. **在 GitHub 创建新仓库**
   - 访问 https://github.com/new
   - 仓库名称：`lifi-carnival-landing`（或你喜欢的名字）
   - 选择 Public 或 Private
   - **不要**勾选 "Initialize with README"
   - 点击 "Create repository"

2. **连接本地仓库到 GitHub**
   ```bash
   # 添加远程仓库（替换 YOUR_USERNAME 为你的 GitHub 用户名）
   git remote add origin https://github.com/YOUR_USERNAME/lifi-carnival-landing.git
   
   # 推送到 GitHub
   git branch -M main
   git push -u origin main
   ```

### 步骤 3：在 Vercel 部署

1. **访问 Vercel**
   - 访问 https://vercel.com
   - 使用 GitHub 账号登录

2. **导入项目**
   - 点击 "Add New..." → "Project"
   - 选择刚才创建的 GitHub 仓库 `lifi-carnival-landing`
   - Vercel 会自动检测到 Next.js 项目

3. **配置项目**
   - Framework Preset: Next.js（自动检测）
   - Root Directory: `./`（默认）
   - Build Command: `npm run build`（自动检测）
   - Output Directory: `.next`（自动检测）
   - Install Command: `npm install`（自动检测）

4. **部署**
   - 点击 "Deploy"
   - 等待构建完成（通常 1-2 分钟）

5. **获取预览链接**
   - 部署完成后，Vercel 会提供一个预览链接
   - 格式：`https://lifi-carnival-landing-xxxxx.vercel.app`
   - 这个链接可以分享给任何人查看

### 步骤 4：配置自定义域名（可选）

1. 在 Vercel 项目设置中
2. 进入 "Domains" 页面
3. 添加你的自定义域名
4. 按照提示配置 DNS 记录

## 方法二：使用 Vercel CLI（快速部署）

如果你不想使用 GitHub，可以直接用命令行部署：

```bash
# 安装 Vercel CLI
npm i -g vercel

# 进入项目目录
cd /Users/yael/Downloads/lifi-carnival-landing

# 部署（首次部署会要求登录）
vercel

# 部署到生产环境
vercel --prod
```

部署完成后会得到一个预览链接。

## 常见问题

### Q: 部署后页面显示空白？
A: 检查浏览器控制台错误，可能是路径问题。确保所有资源文件都在 `public/` 目录。

### Q: 构建失败？
A: 检查 Vercel 构建日志，通常是依赖问题。确保 `package.json` 中所有依赖都正确。

### Q: 如何更新部署？
A: 推送新代码到 GitHub，Vercel 会自动重新部署。

### Q: 如何回滚到之前的版本？
A: 在 Vercel 项目页面 → Deployments → 选择之前的版本 → "Promote to Production"
